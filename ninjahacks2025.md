# NinjaHacks 2025 – Submission Guide

## 1️⃣ Hackathon Overview
- **Hackathon:** NinjaHacks 2025 (FTC Team #26014 Ninjabotics)  
- **Dates:** Nov 14 – Nov 21, 2025  
- **Deadline:** Nov 22, 2025 @ 1:00am GMT+3  
- **Format:** Fully online, beginner-friendly  
- **Team Size:** 1–4 participants  

**Goal:** Build a creative, original, and functional project solving a real-world problem in STEM, sustainability, health, or social good.

---

## 2️⃣ Submission Requirements (Devpost)
All submissions must include:

1. **Project Description**
   - Idea explanation  
   - Problem it solves  
   - Technology stack  

2. **Demo Video (2–4 min)**
   - Showcase project functionality, design, and creativity  
   - Highlight the “wow” moments  

3. **Code Repository**
   - GitHub / GitLab public repo  
   - Include README, setup/run instructions, any APIs used  

4. **Team Info**
   - List of members, roles, contributions  

**Optional:**
- Pitch deck (5–10 slides)  
- Documentation: README, API usage, setup instructions  

---

## 3️⃣ Project Planning & Workflow

**Step 1 – Brainstorm & Select Idea (Nov 14–15)**  
- Focus on **beginner-friendly but impactful ideas**  
- Choose between software (app, web, AI) or hardware (robotics, automation)  
- Example categories:  
  - Social good  
  - STEM education  
  - Environmental sustainability  
  - Health & wellbeing  

**Step 2 – Define MVP (Nov 15)**  
- Core features only  
- Minimal working prototype for demo  
- Allocate tasks among team members  

**Step 3 – Build (Nov 15–20)**  
- Daily stand-up check-ins  
- Version control: commit frequently  
- Keep notes of challenges & solutions for submission write-up  

**Step 4 – Test & Refine (Nov 20–21)**  
- Debug, polish UI/UX  
- Ensure demo video workflow is smooth  
- Prepare code repo, documentation  

**Step 5 – Prepare Submission (Nov 21–22)**  
- Draft project description & README  
- Record demo video  
- Collect team info and contributions  
- Optional: pitch deck  

---

## 4️⃣ Demo Video Storyboard (2–4 minutes)

| Time | Content |
|------|---------|
| 0:00–0:20 | Introduce problem: Why it matters and who it affects |
| 0:20–0:40 | Present solution: Brief overview of project and technology |
| 0:40–1:20 | Demo core functionality: Show main features in action |
| 1:20–2:00 | Highlight unique points / WOW moment: What makes it special |
| 2:00–2:40 | Optional: Future vision, scalability, or social impact |
| 2:40–3:00 | Team introduction and closing remarks |

**Tips:**  
- Keep it concise and engaging  
- Use screen recordings, app interaction, or prototypes  
- Emphasize originality and creativity  

---

## 5️⃣ Example Tech Stack
- **Software:** Python, JS, React, Node.js  
- **AI/ML (optional):** OpenAI, TensorFlow, PyTorch  
- **Hardware (optional):** Arduino, micro:bit, FTC robotics kits  
- **Cloud & Hosting:** GitHub Pages, Heroku, Firebase  

---

## 6️⃣ Judging Criteria (Based on Hackathon Overview)
- **Originality:** Unique idea, not pre-built or copied  
- **Functionality:** Does it work as intended?  
- **Creativity:** Design, UX, and implementation innovation  
- **Impact:** Real-world usefulness, social good, or STEM impact  
- **Collaboration:** Clear roles and teamwork demonstrated  

---

## 7️⃣ Submission Checklist
- [ ] Project description on Devpost  
- [ ] Demo video uploaded (unlisted link)  
- [ ] Public code repo ready  
- [ ] Team info included  
- [ ] Optional: Pitch deck + documentation  
- [ ] Proofread all descriptions and instructions  

---

## 8️⃣ Tips for Success
- Focus on **simplicity + clarity**; MVP is key  
- Record challenges & learnings → include in Devpost write-up  
- Highlight social impact or educational value  
- Be respectful, collaborative, and inclusive
